🚀 The Ultimate Markdown Cheatsheet

Welcome! This guide is designed to help you master Markdown (a lightweight markup language for formatting plain text) specifically for GitHub.

🎯 The 20% Essentials (Pareto Principle)

If you only learn these four things, you can write 80% of all GitHub documentation.

1. Headings

Use # followed by a space. The more # symbols, the smaller the heading.

# Heading 1 (HTML <h1>)

## Heading 2 (HTML <h2>)

### Heading 3 (HTML <h3>)

2. Text Styling

Bold: **Text**

Italic: *Text*

Bold & Italic: ***Text***

~~Strikethrough~~: ~~Text~~

3. Lists

Unordered List: Use +, -, or *

Ordered List: Use 1., 2., 3.

Nested List: Indent by 4 spaces.

Task List (Checkboxes):

  [ ]  - [ ] Task

  [x]  - [x] Done

4. Code Blocks

Inline Code: Use a single backtick ` code `

Code Block: Use three backticks with the language name for syntax highlighting (automatic color coding).

  ```
  // This is JavaScript
  console.log("Hello World");
  ```

📊 Tables and Layout

Creating Tables

Use pipes | and dashes - to create columns and rows.

| Header 1 | Header 2 |
| :--- | :---: |
| Left Align | Center Align |
| Text | Text |


Blockquotes

Use the > symbol to indent text for quotes. You can nest them for deeper levels.

This is a blockquote.

This is a nested blockquote.

🧪 Extended Syntax & GitHub Workarounds

Some Markdown features are "Extended Syntax" and may not work on GitHub natively. Use these HTML tags as fallbacks.

Highlighting: Use <mark>Text</mark> (Result: <mark>Text</mark>)

Superscript: Use <sup>Text</sup> (Result: <sup>Text</sup>)

Subscript: Use <sub>Text</sub> (Result: <sub>Text</sub>)

Line Break: End a line with two spaces to create a <br> tag.

Horizontal Line: Use --- or *** on a new line.
